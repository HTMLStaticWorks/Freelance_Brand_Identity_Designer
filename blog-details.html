The Hierarchy of Type: Setting Precise Scales for Web Applications.

Elena Vance

June 15, 2026 • 6 min read

Typography is the structural skeleton of any interface design. When users land on a website, they do not read at first—they scan the visual rhythm. Without a deliberate mathematical relationship between font sizes, your interfaces will feel cluttered and confusing, regardless of the quality of your images.

Setting a typographic scale is the process of using a defined ratio to scale headings sequentially. In AURA's design methodology, I default to the Major Third (1.250) ratio for standard web interfaces and the Perfect Fourth (1.333) for text-dense strategy blogs.

01 / The Typographic Math

A scale begins with a base size—commonly 16px (1rem) for browser viewports. To calculate the next heading tier, you multiply the base size by your ratio. Under a Major Third scale, the progression is calculated as follows:

  • • Base (p): 16.00px
  • • Heading 4 (h4): 20.00px (1.250 x 16)
  • • Heading 3 (h3): 25.00px (1.250 x 20)
  • • Heading 2 (h2): 31.25px (1.250 x 25)
  • • Heading 1 (h1): 39.06px (1.250 x 31.25)

Using strict mathematical progressions prevents ad-hoc font selections. This system ensures that elements relate logically, creating visual harmony that holds up across all screen configurations.

"A systematic typography system is the line between amateur graphics and senior visual architecture."

02 / Implementing Scales in CSS

To avoid page weight and rendering lag, I define these scales directly inside my global stylesheets using CSS custom properties. Using `clamp()` allows us to set min and max bounds for fluid sizing across viewport breaks:

:root {
  --font-base: 1rem;
  --font-scale: 1.250;

  --font-h4: calc(var(--font-base) * var(--font-scale));
  --font-h3: calc(var(--font-h4) * var(--font-scale));
  --font-h2: calc(var(--font-h3) * var(--font-scale));
  --font-h1: clamp(2rem, 5vw, calc(var(--font-h2) * var(--font-scale)));
}

03 / Accessibility Auditing

No design scale is complete without contrast testing. When pairing typography on surface cards, always verify that your light-gray secondary text complies with a minimum contrast ratio of 4.5:1 against the background under WCAG guidelines. In dark mode, choose slightly warmer tones to limit eye fatigue.
